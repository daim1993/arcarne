Build Arcarne as a cinematic, interactive portfolio website for Diaa Eddin Ahmed Ibrahim / DAIM. The website must not feel like a PDF, résumé page, or static brochure. It should feel like entering a creative universe: motion-led, story-driven, spatial, animated, and visually memorable.

The current information should be preserved, but the presentation must be transformed. Instead of listing experience in flat blocks, the new site should convert the biography, services, work history, toolkits, and portfolio into an immersive digital narrative.

## 4. Design Principle: Do Not Build a Traditional PDF Website

The site must avoid:

- Long résumé-style text blocks
- Static timeline rows
- Flat PDF-like cards
- Overloaded education sections
- Generic portfolio grids
- Plain white background with black text
- Basic “About / Work / Contact” layout without interaction
- Scroll fatigue caused by paragraph-heavy content

The site should use:

- Motion-first storytelling
- Scroll-triggered reveals
- Layered 3D depth
- Dynamic typography
- Interactive project previews
- Animated toolkits
- Cinematic transitions
- Case-study storytelling
- Full-screen visual sections
- Spatial navigation
- Micro-interactions on every meaningful action

## 6. Visual Design System

### Art Direction

The visual language should sit between cinematic sci-fi, premium creative studio, and experimental motion lab.

Recommended visual ingredients:

- Dark immersive background
- High-contrast typography
- Neon accent gradients
- Soft volumetric glow
- Glassmorphism panels used sparingly
- Grain texture for cinematic depth
- 3D abstract forms
- Motion trails
- Grid overlays
- Storyboard frame motifs
- Interface HUD details
- Large editorial typography

### Color Direction

Recommended palette:

- Deep black / graphite as the base
- Electric cyan or spectral blue for interaction states
- Violet / magenta for cinematic glow
- Warm gold or amber for premium highlights
- Soft white for readable body text

Avoid:

- Plain white résumé background
- Generic corporate blue
- Overuse of gradients without structure
- Low-contrast text

### Typography

Recommended pairing:

- Hero / display: bold futuristic or editorial grotesk
- Body: clean modern sans-serif with excellent readability
- Accent: monospaced technical type for coordinates, timeline years, project metadata, and tool labels

Typography behavior:

- Hero words can animate letter-by-letter.
- Section titles can reveal through masking.
- Key phrases can distort slightly on scroll or hover.
- Avoid unreadable experimental type for body copy.

## 7. Animation and Interaction System

### Core Animation Stack

Recommended stack options:

- Framer Motion for React-based UI animation
- GSAP + ScrollTrigger for advanced scroll choreography
- Three.js or React Three Fiber for WebGL 3D scenes
- Spline for faster 3D scene creation and embedding
- Rive for interactive vector animations
- Lottie for lightweight motion assets exported from After Effects
- Lenis for smooth scroll behavior

### Key Animation Effects

Hero:

- 3D object reacts to cursor movement
- Parallax layers move at different speeds
- Headline reveals through cinematic masking
- Project thumbnails drift in controlled depth layers

Navigation:

- Magnetic nav links
- Smooth section transitions
- Menu opens as a full-screen animated overlay
- Active section indicator moves like a scanning beam

Project Cards:

- Hover starts muted video preview
- Cards tilt in 3D based on cursor position
- Background aura changes based on project category
- Click transition expands card into case-study page

Timeline:

- Scroll activates nodes
- Job cards animate from particles into readable panels
- Location lines draw across a world map or abstract grid

Toolkit:

- Tools orbit capability clusters
- Hover pauses orbit and reveals usage
- Selected tools connect to related projects

Contact:

- CTA glows subtly
- Cursor trail becomes more concentrated near contact buttons
- Final send/contact interaction triggers a micro animation

### Motion Rules

- Motion should enhance story, not distract from it.
- Every animation needs a reason: reveal, guide, reward, explain, or transition.
- Use fast micro-interactions and slower cinematic transitions.
- Avoid animating all paragraphs at the same speed.
- Use `prefers-reduced-motion` for accessibility.
- Keep heavy 3D scenes optional or lazy-loaded.

## 10. Technical Build Recommendation

### Preferred Build

Use a modern React-based stack if the goal is a custom high-impact site.

Recommended:

- Next.js
- TypeScript
- Tailwind CSS
- Framer Motion
- GSAP
- Three.js / React Three Fiber
- Spline embeds where faster 3D production is needed
- Sanity or Contentful CMS for project updates
- Vercel deployment

### Framer-Based Alternative

If speed and visual editing are more important than full custom engineering:

- Build in Framer
- Use Framer interactions for standard motion
- Embed Spline scenes for 3D
- Use Lottie / Rive for lightweight animations
- Keep custom code components for advanced hero, project cards, and cursor effects

### Performance Requirements

- Target Lighthouse performance above 85 on desktop
- Keep mobile experience fast and simplified
- Lazy-load videos and WebGL scenes
- Use compressed WebM / MP4 for previews
- Use AVIF / WebP for images
- Do not autoplay heavy video with audio
- Add fallback images for 3D scenes
- Respect reduced-motion preferences
- Keep first load under control by delaying non-critical 3D assets

## 11. Accessibility and UX Requirements

- All text must remain readable on dark backgrounds.
- Do not rely only on hover because mobile users cannot hover.
- Provide keyboard-accessible navigation.
- Use semantic HTML for headings, sections, buttons, and links.
- Keep CTA labels clear.
- Provide reduced-motion fallback.
- Ensure videos are muted by default and controllable.
- Avoid excessive scroll hijacking.
- Maintain strong contrast for body copy.

## 12. Mobile Experience

The mobile version should not simply shrink the desktop design.

Mobile priorities:

- Keep the hero cinematic but lighter.
- Replace heavy 3D scenes with optimized video loops or static fallback renders.
- Use swipeable project cards.
- Turn the toolkit galaxy into horizontal grouped chips.
- Simplify timeline into stacked animated story cards.
- Keep contact CTA sticky near the bottom after the user reaches the work section.
